Definición de Requisitos

Requisitos Funcionales

  • Registro o Inicio de Sesión: Los usuarios podrán registrarse y gestionar su cuenta.
    • Subfunción: Permitir recuperación de contraseñas y autenticación de dos factores.
  • Creacion y gestión de tareas: Los usuarios podrán crear, editar y eliminar tareas.
    • Subfunción: Asignar fechas de vencimiento y etiquetas a las tareas.
  • Asignación de tareas a usuarios: Permitir que los usuarios asignen tareas a otros miembros del equipo.
  • Comentarios y colaboraciones en tareas: Los usuarios podrán dejar comentarios en las tareas para fomentar la colaboración.
  • Notificaciones y recordatorios: Enviar notificaciones sobre próximas fechas de vencimiento y cambios en las tareas.
  • Reportes de productividad: Los usuarios recibirán notificaciones por correo electrónico sobre el estado de su pedido.

Requisitos No Funcionales

  • Usabilidad: La interfaz debe ser intuitiva y fácil de navegar, facilitando la gestión de tareas.
  • Rendimiento: La aplicación debe ser capaz de manejar múltiples usuarios simultáneamente, manteniendo tiempos de respuesta óptimos.
  • Seguridad: Asegurar que los datos de los usuarios y las tareas estén protegidos.
  • Disponibilidad: La plataforma debe estar disponible 24/7, con mínimo tiempo de inactividad.
  • Escalabilidad: Capacidad para manejar un crecimiento en la base de usuarios y tareas sin degradar el rendimiento.
  • Compatibilidad: La aplicación debe funcionar en diferentes dispositivos y navegadores.
  • Mantenibilidad: El código debe estar bien documentado para facilitar futuras actualizaciones.

Diseño de la Arquitectura

Frontend: Desarrollado en React para una interfaz para una interfaz responsiva y amigable.

Backend: con Node.js con una API REST para manejar la gestión de usuarios y tareas.

Base de datos: MongoDB para almacenar información de usuarios y tareas.

Integración con calendarios: APIs para conectar con Google Calendar y Outlook.

Backend

  • Gestion de tareas

Implementación

  • Configurar la API REST para la gestión de usuarios y tareas.
  • Crear una interfaz atractiva con React, incluyendo vistas para tareas pendientes y completadas.
  • Integrar la funcionalidad de notificaciones y recordatorios.

Pruebas

  • Realizar pruebas de carga para evaluar la respuesta de la plataforma bajo múltiples usuarios concurrentes.
  • Testear la funcionalidad de gestión de tareas y comentarios en diferentes dispositivos y navegadores.

Despliegue y Mantenimiento

  • Desplegar la aplicación en AWS para garantizar escalabilidad, con opciones de respaldo y recuperación.
  • Monitorear el rendimiento de la aplicación utilizando CloudWatch, y establecer ciclos de mantenimiento y actualización periódicos.
